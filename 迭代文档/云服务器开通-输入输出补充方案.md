# 云服务器开通（私有云）输入输出补充方案

> 状态：待开发定稿  
> 适用服务：`cloud-vm-private`（云服务器开通-私有云）  
> 关联页面：服务目录（运营端）、服务申请（交付门户）、订单详情（双端）、交付结果录入（运营端）

---

## 一、需求确认结论

| 序号 | 问题 | 结论 |
|------|------|------|
| 1 | VPC 来源 | 当前固定枚举，后续版本对接云管平台动态拉取 |
| 2 | 服务器角色定义 | 平台统一定义，不允许各团队自定义 |
| 3 | 凭证文档交付形式 | 保留 3 种可选：①文本直接录入 ②上传文档链接 ③对接密钥系统链接 |
| 4 | 磁盘规格拆分 | 需要拆分「系统盘」和「数据盘」两个独立选择项 |
| 5 | 安全组/ACL | 选填。用户不填时，交付工程师根据「服务器角色」自动分配默认安全组 |

## 二、输入 Schema 补充设计

### 2.1 完整输入字段清单

| 序号 | key | label | type | required | options / placeholder | 联动规则 |
|------|-----|-------|------|----------|----------------------|----------|
| 1 | `vpc` | 所属 VPC | select | **是** | 见 2.2 VPC 枚举表 | — |
| 2 | `serverRole` | 服务器角色 | select | **是** | 见 2.3 角色枚举表 | 决定默认安全组（见 2.5） |
| 3 | `spec` | 资源规格 | select | **是** | 2C4G / 4C8G / 8C16G / 16C32G | — |
| 4 | `os` | 操作系统 | select | **是** | CentOS 7.9 / Ubuntu 22.04 / Rocky 9 / Windows 2019 | — |
| 5 | `systemDisk` | 系统盘规格 | select | **是** | 见 2.4 磁盘枚举表 | — |
| 6 | `dataDisk` | 数据盘规格 | select | 否 | 见 2.4 磁盘枚举表，默认「无」 | — |
| 7 | `count` | 数量 | integer | **是** | 1 ~ 50（max 根据角色限制，见 2.6） | — |
| 8 | `securityGroup` | 安全组/ACL | text | 否 | 如：sg-web-prod | 不填时按角色分配默认安全组 |
| 9 | `appGroup` | 应用资源分组 | select | 否 | 见 2.7 资源分组枚举表 | 用于第 4 步自动挂载标签 |
| 10 | `purpose` | 用途说明 | text | 否 | 请简要说明用途 | — |

### 2.5 安全组/ACL 联动规则

```
IF securityGroup 为空:
    自动填充默认值 = serverRole 对应的默认安全组
    并在表单中以「置灰提示」展示，如：
    「未指定，将使用默认安全组：sg-web-default」
ELSE:
    使用用户输入值
```

### 2.6 数量上限联动规则

```
count 的 max 值 = serverRole 对应的数量上限
前端表单根据角色选择动态限制数量输入框的 max 属性
```

## 三、输出 Schema 补充设计

### 3.1 完整输出字段清单

输出 Schema 用于两个场景：
1. **交付结果录入**：运营中心工程师按 Schema 字段录入实际交付结果
2. **订单详情展示**：交付门户用户查看时，按 Schema 字段展示交付信息

| 分类 | key | label | type | required | 说明 |
|------|-----|-------|------|----------|------|
| **资产信息** | `assetId` | 资产编号 | text | 是 | 如 `BJ-VM-2026-00421` |
| | `location` | 物理位置 | text | 是 | 如 `北京亦庄 DC2 / 机柜 A12-03` |
| | `rackUnit` | 机位 | text | 否 | 如 `U24` |
| **网络信息** | `hostname` | 主机名 | text | 是 | — |
| | `ip` | IP 地址 | text | 是 | — |
| | `subnet` | 子网掩码 | text | 是 | 如 `255.255.255.0` |
| | `gateway` | 网关 | text | 是 | 如 `10.20.30.1` |
| | `vlan` | VLAN | text | 是 | 如 `VLAN-2030` |
| **规格信息** | `cpu` | CPU | text | 是 | 如 `4 vCPU` |
| | `memory` | 内存 | text | 是 | 如 `8 GB` |
| | `systemDisk` | 系统盘 | text | 是 | 如 `50 GB SSD` |
| | `dataDisk` | 数据盘 | text | 否 | 如 `200 GB SSD`，无则为「—」 |
| | `osActual` | 操作系统（实际） | text | 是 | 如 `CentOS 7.9 x86_64` |
| **凭证信息** | `credentialType` | 凭证类型 | select | 是 | 见 3.2 |
| | `credentialValue` | 凭证内容 | textarea | 条件必填 | 见 3.2 |
| | `credentialUrl` | 凭证文档链接 | text | 条件必填 | 见 3.2 |
| **集成服务** | `pamStatus` | PAM 堡垒机 | text | 否 | 如 `active` + 链接 |
| | `monitorStatus` | 监控 | text | 否 | 如 `active` + Grafana 链接 |
| | `loggingStatus` | 日志 | text | 否 | 如 `active` + Kibana 链接 |
| | `backupStatus` | 备份 | text | 否 | 如 `每日 02:00 增量备份` |
| | `securityStatus` | 安全 | text | 否 | 如 `Agent v3.2 已安装` |
| **交付验收** | `connectivityTest` | 连通性测试结果 | textarea | 否 | ping / ssh / 端口测试摘要 |
| | `deliveredAt` | 交付时间 | text | 是 | 如 `2026-06-01 14:30:00` |
| | `deliveredBy` | 交付工程师 | text | 是 | — |

### 3.2 凭证信息字段设计（3 种形式可选）

| credentialType | credentialValue | credentialUrl | 适用场景 |
|----------------|-----------------|---------------|----------|
| `plaintext` | 填写密码 / SSH Key 文本 | 空 | 简单场景，内部信任环境 |
| `document` | 空或备注 | 上传文档的下载链接 | 正式交付，提供标准化文档 |
| `vault` | 空或备注 | 密钥管理系统访问链接 | 高安全要求，敏感凭证不进系统 |

**交互规则：**
```
IF credentialType == "plaintext":
    credentialValue 必填，credentialUrl 隐藏
    展示时增加「复制」按钮，30 秒后自动隐藏或需二次验证查看

IF credentialType == "document":
    credentialUrl 必填，credentialValue 隐藏
    展示时渲染为可点击下载链接

IF credentialType == "vault":
    credentialUrl 必填，credentialValue 隐藏
    展示时渲染为「前往密钥系统查看」外部链接
```

### 4.3 交互细节

| 交互项 | 规则 |
|--------|------|
| 表单初始化 | 自动填充申请时的 `spec`、`os`、`systemDisk`、`dataDisk`、`count` 到输出表单的对应字段，工程师可修改 |
| 资产编号 | 建议格式 `BJ-VM-YYYY-NNNNN`，前端做格式校验提示，不做强制阻断 |
| IP 地址 | 格式校验，且需校验是否属于所选 VPC 的网段（后续对接网络平台时自动校验） |
| 凭证内容 | `textarea`，密码类型可切换「显示/隐藏」。`plaintext` 模式下必填 |
| 保存草稿 | 允许部分字段为空，保存后不推进订单状态 |
| 确认交付完成 | 所有 `required: true` 字段必须已填，提交后订单状态变为 `completed`，触发用户通知 |
| 批量录入 | 数量 > 1 时，支持「添加下一台」分页签录入，或「批量导入」Excel/CSV |

**凭证查看安全规则：**
- 默认折叠，点击「查看凭证」后展开
- `plaintext` 类型：首次查看需二次验证（如短信验证码或登录密码确认），30 秒后自动隐藏
- `document` 类型：直接展示下载链接
- `vault` 类型：展示「前往密钥系统查看」外部链接

### 5.2 运营中心（工程师侧）

在运营中心订单详情中，除展示交付信息外，增加：
- 「编辑交付结果」按钮（已完成后仍可修正）
- 「重新交付」按钮（异常情况）
- 交付执行步骤时间线（见 5.3）

### 5.3 交付执行步骤时间线

在订单详情中增加与实际 5 步处理对应的时间线：

```
申请受理 ──→ 方案确认 ──→ 交付执行 ──→ 验证确认 ──→ 交付完成
                              │
                              ▼
                    ① 镜像准备          [✓ 2026-06-01 09:00]
                    ② VPC/子网/安全组配置 [✓ 2026-06-01 09:15]
                    ③ 创建实例           [✓ 2026-06-01 09:30]
                    ④ 资源分组挂载        [✓ 2026-06-01 09:45]
                    ⑤ 连通性测试与交付     [✓ 2026-06-01 10:00]
```

每个步骤可展开查看详细日志或配置信息。

| 场景 | 处理方案 |
