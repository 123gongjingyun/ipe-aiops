## 一、项目背景与改造原则

### 1.1 目标

将仿仿生产环境（`10.66.220.250:41231`）的核心能力——**资源配置管理**、**智能配置推荐**、**角色权限体系**、**审批流程管理**——融入现有 IPE/AIOps 平台，实现平台对仿生产环境的替代能力，同时保留现有服务目录和 AIOps 交付能力。

### 1.2 改造原则

| 原则 | 说明 |
|------|------|
| **服务目录保留** | 现有85个服务（81原子+4组合）及 Portal/Center 入口保持不变 |
| **配置依附服务** | 不新建独立模块，配置数据字典依附于原子服务，配置模板依附于组合服务 |
| **最小化侵入** | 现有服务规格、组合服务、工单流程全部保留，只做增量扩展 |
| **不重复建设** | 充分利用现有"全部服务"编辑页面、SLA管理页面，在其基础上增量扩展 |
| **分期实施** | 本期聚焦核心能力（配置+角色+审批），批量申请等能力留待二期 |

### 3.2 关键设计决策

| 决策项 | 决策 | 理由 |
|--------|------|------|
| **配置是否独立管理** | ❌ 否，配置依附于服务 | 最小化改造，不新增独立模块 |
| **服务编辑页面** | 复用现有"全部服务"编辑弹窗，在其内增加配置数据字典区域 | 避免重复建设 |
| **组合模板管理** | 复用现有组合服务编辑弹窗，在其内增加配置模板区域 | 避免重复建设 |
| **SLA管理** | ✅ 完全保留现有SLA管理页面 | 已满足需求，但需要和AI推荐关联 |
| **角色/用户管理位置** | 放到"设置"页面下作为二级标签 | 简洁导航，不占用一级菜单 |
| **审批流程配置** | 独立页面，支持可视化流程图 | 审批流程复杂，需独立配置界面 |
| **批量申请** | ⏳ 本期不做 | 最小化范围，留二期 |

## 五、页面改造方案

### 5.1 原子服务申请（/apply-service/:serviceId）

**现有流程**：需求描述 → AI推荐 → 提交成功（3步）

**改造后**：

```
Step 0: 需求描述（现有 inputSchema 字段：规格/OS/数量/用途等）

Step 1: 需求参数确认（新增）
  ├─ SLA 级别 [金牌/银牌/铜牌]  ← 可覆盖服务默认SLA
  ├─ 部署环境 [开发/UAT/生产/压测]
  ├─ 预期规模 [日活<1,000 / 1K-1W / 1W-10W / >10万]
  ├─ 系统编号
  ├─ 系统名称
  ├─ 模块名称
  └─ 担当

Step 2: AI 智能推荐配置（改造核心）
  ├─ 根据 SLA + 环境 + 规模匹配最佳 configProfile
  ├─ 展示推荐配置详情卡片
  ├─ 风险校验：
  │   例：用户选"金牌SLA+仿生产环境"，inputSchema里选了"单机"
  │   → 系统⚠️提示并自动推荐"主从架构"配置
  └─ 允许用户手动切换其他配置（不匹配的标红提示）

Step 3: 提交成功
```

### 5.2 组合服务申请（/apply/:comboId）

**互联网应用部署**（现有5步Wizard）：

```
Step 0: 应用信息（现有）
  ├─ 补充：系统编号、模块名称、担当、SLA级别确认

Step 0.5: 选择配置模板（新增）
  ├─ 小型方案：低配容器 + 单机MySQL + 单机Redis + 基础网络
  ├─ 中型方案：中配容器 + 主从MySQL + 哨兵Redis + 标准网络
  └─ 大型方案：高配容器 + 集群MySQL + 集群Redis + 高级网络 + CDN/WAF
  
  选择后：Step 1-3 默认值自动联动

Step 1: 计算资源（AI推荐，基于模板预设，可覆盖）
Step 2: 数据库（AI推荐，基于模板预设，可覆盖）
Step 3: 网络发布（可覆盖默认值）
Step 4: 确认提交
```

**其他组合服务**（现有3步Wizard）：

```
Step 0: 需求描述（现有 inputSchema）
  ├─ 补充：系统编号、模块名称、担当、SLA级别确认

Step 0.5: 选择配置模板（新增）
Step 1: AI推荐（基于模板生成编排方案）
Step 2: 提交成功
```

## 七、智能推荐算法（核心亮点）

### 7.1 原子服务配置推荐

```typescript
function recommendAtomicConfig(
  service: AtomicServiceSpec,
  slaLevel: "gold" | "silver" | "bronze",
  env: string,
  userInputs: Record<string, string>,
  profiles: ServiceConfigProfile[]
): RecommendationResult {
  const warnings: string[] = [];

  // 1. 环境过滤
  const envMatched = profiles.filter(p => p.suitableEnvs.includes(env));

  // 2. SLA过滤
  const slaMatched = (envMatched.length > 0 ? envMatched : profiles)
    .filter(p => p.suitableSlaLevels.includes(slaLevel));

  // 3. 风险校验：金牌+生产+单机 → 风险
  const haMode = userInputs["haMode"];
  if (slaLevel === "gold" && env === "prod" && haMode === "单机") {
    warnings.push("⚠️ 金牌SLA仿生产环境不建议使用单机架构，已自动推荐高可用配置");
  }

  // 4. 评分排序
  const scored = (slaMatched.length > 0 ? slaMatched : profiles).map(p => {
    let score = 0;
    if (p.suitableSlaLevels.includes(slaLevel)) score += 10;
    if (p.suitableEnvs.includes(env)) score += 5;
    if (p.isDefault) score += 3;
    if (slaLevel === "gold" && env === "prod" && p.details.architecture !== "单节点") score += 5;
    return { profile: p, score };
  });

  scored.sort((a, b) => b.score - a.score);

  return {
    recommended: scored[0]?.profile,
    alternatives: scored.slice(1, 4).map(s => s.profile),
    warnings
  };
}
```

### 7.2 推荐示例

| 用户输入 | 系统行为 |
|---------|---------|
| SLA金牌 + 仿生产环境 + 日活>10万 | 推荐"配置C-高性能主从"（2节点 8C16G），禁用单机配置 |
| SLA金牌 + 仿生产环境 + 用户手动选"配置A-小型单机" | 弹窗："金牌SLA仿生产环境不建议单机，推荐升级到主从架构"，提供一键替换 |
| SLA银牌 + UAT环境 + 日活<3000 | 推荐"配置A-小型"（1节点 2C4G），允许升级中型 |
| SLA铜牌 + 开发环境 | 推荐最低配，所有配置可选 |
| 架构图+用户手动选“单机/主从” | 根据架构图里高可用，推荐将“单机/主从”调整为 **“主从+MHA”**，以对齐架构设计，消除单点隐患 |
