# IPE/AIOps 现网升级执行命令清单

## 1. 适用场景

本清单适用于当前 `www.getpre.cn` 的真实现网形态：

- 平台首页：`https://www.getpre.cn/`
- 服务门户：`https://www.getpre.cn/portal/`
- 运营中心：`https://www.getpre.cn/center/`
- 前端容器：`presales-frontend`
- 宿主机真实发布目录：`/home/deploy/project-myway-platform/`
- Portal 生效目录：`/home/deploy/project-myway-platform/portal/`
- Center 生效目录：`/home/deploy/project-myway-platform/center/`
- 容器内只读映射目录：`/usr/share/nginx/platform/`
- Nginx 配置源文件：`/opt/presales-platform/frontend/nginx.conf`

重要结论：

- 不要再发布到 `/usr/share/nginx/html/portal/` 或 `/usr/share/nginx/html/center/`。
- 不要依赖 `docker cp` 到容器内 `/usr/share/nginx/platform/...`，该目录是宿主机目录的只读挂载。
- 正确方式是覆盖宿主机 `/home/deploy/project-myway-platform/portal/` 和 `/home/deploy/project-myway-platform/center/`。

## 2. 本地构建与上传

进入项目：

```bash

```

跑测试：

```bash

```


如需要共享演示数据，同时上传同步服务脚本：

## 6. 可选：启动共享演示数据服务

如果只是让别人打开页面看静态效果，可以跳过本节。

如果希望不同访问者看到同一批演示工单和交付资产，需要启动同步服务。

同步服务默认监听 `0.0.0.0:3011`，便于 `presales-frontend` 容器内 Nginx 通过宿主机网关 `172.18.0.1:3011` 访问。

准备目录：

```bash
mkdir -p /home/deploy/scripts /home/deploy/.dev-data
```

启动服务：

```bash
cd /home/deploy/scripts
nohup node ./dev-orders-sync-server.mjs > /home/deploy/orders-sync.log 2>&1 &
```

验证本机服务：

```bash
ps -ef | grep dev-orders-sync-server | grep -v grep
curl http://127.0.0.1:3011/api/dev/orders-sync
cat /home/deploy/orders-sync.log
```

说明：

- 返回 `{"orders":null,"updatedAt":null}` 表示服务已启动但还没有共享数据。
- `curl` 连接失败时，优先检查脚本是否上传到 `/home/deploy/scripts/dev-orders-sync-server.mjs`。
- 日志中如果显示 `listening on http://0.0.0.0:3011/api/dev/orders-sync`，属于正常结果。

## 7. 可选：配置 Nginx 同源代理

修改宿主机配置文件：

```bash
vi /opt/presales-platform/frontend/nginx.conf
```

在 `server_name www.getpre.cn;` 对应的 443 server 中增加：

```nginx
location /api/dev/orders-sync {
  proxy_pass http://172.18.0.1:3011/api/dev/orders-sync;
  proxy_http_version 1.1;
  proxy_set_header Host $host;
  proxy_set_header X-Real-IP $remote_addr;
  proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
  proxy_set_header X-Forwarded-Proto $scheme;
}
```

检查并重载：

```bash
docker exec presales-frontend nginx -t
docker exec presales-frontend nginx -s reload
```

验证公网同源接口：

```bash
curl -ks https://www.getpre.cn/api/dev/orders-sync
```

## 10. 升级完成后的下一步

前端替换完成后，建议按顺序推进：

1. 确认共享演示数据是否满足“不同访问者打开即可看到效果”。
2. 将 `orders-sync` 从临时开发同步服务演进为正式 API + DB。
3. 将订单、模板、字段字典、配置档案、交付资产迁到数据库。
4. 接入真实 ITSM 审批、正式 CMDB 同步和自动交付系统。
