# Center-菜单权限矩阵-2026-07-20

## 一、目的

本矩阵用于收口 `Center` 侧登录后可见菜单与页面守卫口径，作为后续：

1. 左侧导航渲染过滤
2. 路由级权限校验
3. 管理侧默认首页判断

的统一依据。

本矩阵只定义 **菜单级与页面级访问权限**，不覆盖全部动作级按钮权限。

## 七、默认首页建议

建议按角色设置默认落地页：

| 角色 | 默认首页 |
| --- | --- |
| `platform-admin` | `/` |
| `delivery-engineer` | `/orders` |
| `reviewer` | `/orders` |
| `ops` | `/ops-integration` |
| `security-admin` | `/ops-integration` |

当用户具有多个角色时，建议优先级：

1. `platform-admin`
2. `delivery-engineer`
3. `ops`
4. `security-admin`
5. `reviewer`

如果一个用户只有 `applicant`，建议默认不开放 Center。

---

## 八、第一阶段实现建议

第一阶段建议只落下面这些点：

1. `center-layout` 按 `menuKey` 过滤导航
2. `App.tsx` 中为所有业务路由补 `RequireAuth`
3. 对敏感管理页补 `RequireMenuAccess`
4. 登录成功后按角色跳默认首页

建议第一阶段重点保护页面：

1. `/settings`
2. `/service-catalog`
3. `/service-ledger`
4. `/ops-integration`

---

## 九、注意事项

1. `settings` 默认只对 `platform-admin` 开放
2. `service-catalog/templates` 是跳转到设置页模板 tab，权限应归 `menu.center.settings`
3. `ops-integration` 及其扩展页应作为运维、安全、交付的共享能力组，不单独再拆一批一级角色
4. Center 不建议对 `applicant` 第一阶段开放，避免用户入口混乱
