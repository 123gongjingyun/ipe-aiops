# 现网证书更新说明

## 1. 文档目的

本文记录 `www.getpre.cn / getpre.cn` 现网 HTTPS 证书的实际挂载路径、Nginx 引用关系、标准替换步骤、校验命令与回滚方式，供后续更新证书时直接复用。

---

## 2. 现网证书结构

### 2.1 容器与挂载关系

现网前端容器：

- `presales-frontend`

证书宿主机目录：

- `/opt/presales-platform/getpre.cn_nginx`

容器内挂载目标：

- `/etc/nginx/certs`

当前已确认的挂载关系：

```text
/opt/presales-platform/getpre.cn_nginx -> /etc/nginx/certs
```

这意味着：

- 不需要进入容器内手工替换证书文件
- 正确方式是在宿主机替换 `/opt/presales-platform/getpre.cn_nginx` 下的证书文件
- 容器内会自动看到更新后的文件

## 3. 当前 Nginx 实际引用的证书文件

根据 `nginx -T` 结果，当前 HTTPS server 使用的是：

```nginx
ssl_certificate /etc/nginx/certs/getpre.cn_bundle.crt;
```

对应宿主机实际文件就是：

- `/opt/presales-platform/getpre.cn_nginx/getpre.cn_bundle.crt`

目录现状：

- `getpre.cn.csr`

说明：

- Nginx 当前真正使用的是 `.crt + .key`
- `getpre.cn_bundle.pem` 当前未直接被 Nginx 引用，但建议与 `.crt` 保持一致更新，便于后续维护

## 5. 标准更新步骤

### 5.1 进入证书目录

```bash
cd /opt/presales-platform/getpre.cn_nginx
```

### 5.2 备份旧证书

```bash
BACKUP_DIR="backup_$(date +%Y%m%d_%H%M%S)"
mkdir -p "$BACKUP_DIR"

cp getpre.cn_bundle.crt "$BACKUP_DIR/"
cp getpre.cn.key "$BACKUP_DIR/"
cp getpre.cn_bundle.pem "$BACKUP_DIR/" 2>/dev/null || true
```

### 5.3 用新证书覆盖正式文件

如果新申请的证书文件已经上传到服务器，例如：

- `/home/deploy/new-cert/fullchain.pem`
- `/home/deploy/new-cert/privkey.key`

可以执行：

```bash
cp /home/deploy/new-cert/fullchain.pem /opt/presales-platform/getpre.cn_nginx/getpre.cn_bundle.crt
cp /home/deploy/new-cert/fullchain.pem /opt/presales-platform/getpre.cn_nginx/getpre.cn_bundle.pem
cp /home/deploy/new-cert/privkey.key /opt/presales-platform/getpre.cn_nginx/getpre.cn.key
```

如果新证书本身就是：

- `getpre.cn_bundle.crt`
- `getpre.cn.key`

则直接覆盖对应文件即可。

## 6. 替换后校验步骤

### 6.1 先校验证书和私钥是否匹配

```bash
openssl x509 -noout -modulus -in /opt/presales-platform/getpre.cn_nginx/getpre.cn_bundle.crt | openssl md5
openssl rsa  -noout -modulus -in /opt/presales-platform/getpre.cn_nginx/getpre.cn.key | openssl md5
```

要求：

- 两条命令输出必须一致

如果不一致：

- 说明证书与私钥不是一对
- 不要 reload Nginx

### 6.2 检查 Nginx 配置

```bash
docker exec presales-frontend nginx -t
```

要求：

- `syntax is ok`
- `test is successful`

### 6.3 重载 Nginx

```bash
docker exec presales-frontend nginx -s reload
```

## 7. 线上验证命令

### 7.1 校验证书有效期与颁发者

`getpre.cn`：

```bash
echo | openssl s_client -connect getpre.cn:443 -servername getpre.cn 2>/dev/null | openssl x509 -noout -dates -issuer -subject
```

`www.getpre.cn`：

```bash
echo | openssl s_client -connect www.getpre.cn:443 -servername www.getpre.cn 2>/dev/null | openssl x509 -noout -dates -issuer -subject
```

### 7.2 校验 HTTPS 访问

```bash
curl -Iv https://www.getpre.cn
curl -Iv https://www.getpre.cn/portal/
curl -Iv https://www.getpre.cn/center/
```

要求：

- `SSL certificate verify ok.`
- 返回 `HTTP/2 200`

## 8. 当前现网 Nginx 证书引用位置

当前配置来源文件：

- `/opt/presales-platform/frontend/nginx.conf`

容器内 `nginx -T` 已确认以下 server 使用同一套证书：

### 8.1 `getpre.cn`

```nginx
server {
  listen 443 ssl http2;
  server_name getpre.cn;

  ssl_certificate /etc/nginx/certs/getpre.cn_bundle.crt;
  ssl_certificate_key /etc/nginx/certs/getpre.cn.key;
}
```

### 8.2 `www.getpre.cn`

```nginx
server {
  listen 443 ssl http2;
  server_name www.getpre.cn;

  ssl_certificate /etc/nginx/certs/getpre.cn_bundle.crt;
  ssl_certificate_key /etc/nginx/certs/getpre.cn.key;
}
```

因此当前一套证书同时覆盖：

- `getpre.cn`
- `www.getpre.cn`
- `Portal`
- `Center`

## 9. 回滚步骤

如果新证书替换后出现：

- `nginx -t` 不通过
- 证书链异常
- 域名不匹配
- HTTPS 无法访问

可以立即回滚：

```bash
cd /opt/presales-platform/getpre.cn_nginx

cp "$BACKUP_DIR/getpre.cn_bundle.crt" ./getpre.cn_bundle.crt
cp "$BACKUP_DIR/getpre.cn.key" ./getpre.cn.key
cp "$BACKUP_DIR/getpre.cn_bundle.pem" ./getpre.cn_bundle.pem 2>/dev/null || true

docker exec presales-frontend nginx -t
docker exec presales-frontend nginx -s reload
```

回滚后再次验证：

```bash
curl -Iv https://www.getpre.cn
curl -Iv https://www.getpre.cn/portal/
curl -Iv https://www.getpre.cn/center/
```

## 10. 维护建议

### 10.1 证书更新前

建议固定先做三件事：

1. 备份旧证书
2. 校验新证书与私钥匹配
3. 先 `nginx -t` 再 `reload`

### 10.2 证书更新后

建议固定做三类验证：

1. `openssl s_client` 校验有效期
2. `curl -Iv` 校验 HTTPS 访问
3. 验证 Portal / Center 路径是否正常

### 10.3 文档口径

后续所有现网证书相关说明，建议统一使用以下真实口径：

- 证书宿主机目录：`/opt/presales-platform/getpre.cn_nginx`
- 容器内挂载目录：`/etc/nginx/certs`
- 当前正式证书文件：`getpre.cn_bundle.crt`
- 当前正式私钥文件：`getpre.cn.key`

避免后续误以为证书在容器内手工管理。
