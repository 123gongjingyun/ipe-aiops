# 阶段 1：导出体系升级任务清单

> 文档日期：2026-07-17  
> 适用项目：`project-mywayaiops-refactor-V2`  
> 阶段目标：先完成 `HTML / PDF / Excel` 评审材料导出体系升级，为后续数据库、登录权限、配置中心改造提供稳定输出面

---

## 1. 本阶段目标

当前阶段只做一件核心事：

- 把现有“能导 Excel”的状态，升级成“能生成正式评审材料”的导出体系

本阶段完成后，应具备：

1. `HTML 评审版` 在线预览能力
2. `PDF 打印版` 审阅输出能力
3. `Excel 数据版` 结构化导出能力
4. `审批意见` 占位与补录后二次导出能力
5. `评审交流话术建议` 的导出专用呈现能力

---

## 2. 约束与口径

### 2.1 字段冻结约束

本阶段不修改当前已和用户确认过的常见资源申请字段口径：

- 不改字段名
- 不改字段含义
- 不改字段分组
- 不改字段顺序

允许做的只是：

- 导出结构优化
- 展示布局优化
- 审阅体验优化
- 占位提示优化

### 2.2 导出结构固定口径

导出统一结构固定为：

1. 用户需求
2. 申请信息
3. 评审概要概览
4. 评审交流话术建议
5. 审批意见

Excel 固定结构为：

- `用户需求` sheet
- `申请信息` sheet
- 每个组件单独 sheet
- `评审概要概览` sheet
- `评审交流话术建议` sheet
- `审批意见` sheet

### 2.3 用户需求规则

1. `用户需求` 整块按必填区处理
2. 粉色字段为项目组填写区
3. 除 `背景` 和 `网络需求-用户访问方式` 外，不涉及项允许填写 `无`
4. `背景` 与 `网络需求-用户访问方式` 不能填写 `无`
5. `重要总原则` 放在 `用户需求` 顶部统一提示，不挂单字段

### 2.4 评审交流话术建议规则

1. 不放在日常填写页主界面
2. 只放在导出材料中
3. HTML 评审页默认折叠
4. 需要有明确展开/收起入口
5. 后续可补 `复制话术` 能力
6. PDF 导出应支持是否包含该内容的开关
