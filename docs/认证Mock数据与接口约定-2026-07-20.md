## 三、统一数据结构

## 3.1 AuthUser

建议 Shared 统一采用下面结构：

```ts
interface AuthUser {
  id: string;
  username: string;
  displayName: string;
  email?: string;
  roleKeys: string[];
  roleLabels: string[];
  effectiveMenuKeys: string[];
  effectiveActionKeys: string[];
  isActive: boolean;
}
```

字段说明：

| 字段 | 含义 | 说明 |
| --- | --- | --- |
| `id` | 用户唯一标识 | 前端与后端统一用户主键 |
| `username` | 登录账号 | 建议与企业账号或工号一致 |
| `displayName` | 展示名称 | 页面头部和用户菜单展示 |
| `email` | 邮箱 | 可选 |
| `roleKeys` | 角色 key 列表 | 允许多角色 |
| `roleLabels` | 角色中文名列表 | 便于页面展示 |
| `effectiveMenuKeys` | 最终菜单权限 | 渲染导航与页面守卫 |
| `effectiveActionKeys` | 最终动作权限 | 按钮级权限校验 |
| `isActive` | 账号是否启用 | 禁用账号登录后应视为不可用 |

## 3.2 AuthSession

建议统一会话结构：

```ts
interface AuthSession {
  accessToken: string;
  currentUser: AuthUser;
  issuedAt: string;
}
```

说明：

1. `issuedAt` 用于调试、刷新会话、问题排查
2. 前端第一阶段可不引入 `refreshToken`
3. 第二阶段如后端需要，可再扩展

## 四、本地存储约定

建议 Shared 统一使用下面 key：

| 存储项 | key |
| --- | --- |
| accessToken | `ipe_auth_access_token` |
| currentUser | `ipe_auth_current_user` |
| sessionMeta | `ipe_auth_session_meta` |

说明：

1. 不建议继续使用过于泛化的 `accessToken`
2. 加 `ipe_auth_` 前缀，避免与其他模块冲突
3. Portal 与 Center 共用同一组 key

## 九、真实接口建议

## 9.1 登录

### 请求

`POST /auth/login`

```json
{
  "username": "zhou.ning",
  "password": "123456"
}
```

### 响应

```json
{
  "success": true,
  "accessToken": "jwt-or-session-token",
  "currentUser": {
    "id": "user-applicant-1",
    "username": "zhou.ning",
    "displayName": "周宁",
    "email": "zhou.ning@getpre.cn",
    "roleKeys": ["applicant"],
    "roleLabels": ["申请人"],
    "effectiveMenuKeys": ["menu.portal.home"],
    "effectiveActionKeys": ["action.portal.request.create"],
    "isActive": true
  }
}
```

## 9.2 注册

第一阶段建议保留接口能力，但不默认开放前端入口。

### 请求

`POST /auth/register`

```json
{
  "username": "new.user",
  "displayName": "新用户",
  "email": "new.user@getpre.cn",
  "password": "123456"
}
```

### 响应

```json
{
  "success": true,
  "message": "注册成功"
}
```

## 9.3 获取当前用户

### 请求

`GET /auth/me`

Header:

```text
Authorization: Bearer <accessToken>
```

### 响应

```json
{
  "success": true,
  "currentUser": {
    "id": "user-delivery-1",
    "username": "wang.qh",
    "displayName": "王启航",
    "roleKeys": ["delivery-engineer"],
    "roleLabels": ["交付担当"],
    "effectiveMenuKeys": ["menu.center.orders"],
    "effectiveActionKeys": ["action.center.catalog.manage"],
    "isActive": true
  }
}
```

## 9.4 登出

### 请求

`POST /auth/logout`

Header:

```text
Authorization: Bearer <accessToken>
```

### 响应

```json
{
  "success": true
}
```

## 十、前端错误口径建议

建议统一错误文案：

| 场景 | 提示 |
| --- | --- |
| 登录失败 | `账号或密码错误` |
| 用户被禁用 | `当前账号已被停用，请联系管理员` |
| 会话失效 | `登录已失效，请重新登录` |
| 无菜单权限 | `当前账号暂无该页面访问权限` |
| 网络异常 | `无法连接认证服务，请稍后重试` |
